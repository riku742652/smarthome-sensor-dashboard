## Common Feedback Themes

例：

これらを次回の計画で事前に対応できるようにします。

## Task Type Strategies

### 新機能追加
- リサーチドキュメントから既存パターンを特定
- 小さく始めて段階的に拡張
- テストカバレッジを最初から計画

### バグ修正
- 根本原因の特定を最優先
- 再発防止のテストを含める
- 影響範囲を明確に

### リファクタリング
- 既存動作を変更しないことを確認
- 段階的な変更を計画
- 各ステップで動作確認

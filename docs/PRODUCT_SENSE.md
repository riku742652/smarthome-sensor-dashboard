# Product Sense

このドキュメントは、製品の目標、優先順位、意思決定の指針を定義します。

## 製品ビジョン

**快適な室内環境を維持するための、シンプルで直感的な環境モニタリングダッシュボード**

### なぜこの製品を作るのか

- 室内の温度、湿度、CO2濃度は健康と生産性に直接影響する
- Switchbotセンサーは優れているが、公式アプリは十分な可視化機能がない
- 時系列データのトレンドを見ることで、換気や空調の最適化ができる

## ユーザーペルソナ

### プライマリユーザー: 在宅ワーカー

- 自宅で長時間作業する
- 快適な環境で生産性を維持したい
- CO2濃度が高いと集中力が低下することを知っている
- 定期的に換気のタイミングを知りたい

### セカンダリユーザー: 家族の健康を気にする親

- 子供の部屋の環境を監視したい
- 夜間の温湿度が適切か確認したい
- 複数の部屋を比較したい

## 製品原則

### 1. シンプルさを最優先

- 機能を詰め込まない
- ワンスクリーンで完結
- 3秒で現状を把握できる

### 2. データの可視化に集中

- グラフは直感的で美しく
- 快適範囲を視覚的に示す
- トレンドが一目でわかる

### 3. 信頼性

- データは常に最新
- エラーは明確に表示
- オフライン時の挙動も考慮

### 4. パフォーマンス

- ロードは高速
- スムーズなアニメーション
- データ取得は効率的

## 機能の優先順位

### P0（必須 - MVP）

1. センサーデータの表示（現在値）
2. 24時間グラフ
3. 自動更新（1分間隔）

### P1（重要 - Phase 2）

1. 複数の時間範囲（1時間、24時間、1週間）
2. ダークモード
3. レスポンシブデザイン

### P2（あると良い - 将来）

1. 複数センサーのサポート
2. カスタムアラート
3. データエクスポート
4. 統計情報（平均、最大、最小）

### P3（nice to have）

1. 天気情報との連携
2. 履歴データの長期保存
3. AIによる最適化提案

## 意思決定の指針

### 新機能を追加するか迷ったら

以下の質問に答える：

1. **ユーザーの主要な問題を解決するか？**
   - Yes → 検討を続ける
   - No → 却下

2. **シンプルさを損なわないか？**
   - Yes（損なわない） → 検討を続ける
   - No → 却下または簡略化

3. **現在のフェーズの目標に合致するか？**
   - Yes → 実装を計画
   - No → 将来の機能としてバックログへ

### トレードオフの判断

- **速度 vs 機能**: 速度を優先（MVPは最小限の機能）
- **美しさ vs 機能**: 美しさを優先（データ可視化が核心）
- **柔軟性 vs シンプルさ**: シンプルさを優先（設定は最小限）

## 成功指標

### MVP段階

- [ ] デイリーアクティブユーザー: 1人（自分）
- [ ] ダッシュボードの確認頻度: 1日5回以上
- [ ] データ取得の成功率: 95%以上
- [ ] ページロード時間: 3秒以内

### Phase 2

- [ ] 継続利用率: 1週間後も80%
- [ ] モバイルでの利用: 全体の30%以上
- [ ] エラー率: 1%未満

## やらないこと（明示的に）

- ❌ Switchbotデバイスの制御（読み取り専用に集中）
- ❌ ユーザー認証（まずは個人利用）
- ❌ SNS連携
- ❌ 複雑なデータ分析機能
- ❌ リアルタイムアラート（Phase 1では）

## フィードバックループ

### 定期的に評価

- 週次: 使用感の振り返り
- 月次: 成功指標の確認
- 機能追加時: ユーザーテスト（自分）

### 改善サイクル

1. 使用中の痛点を記録
2. 優先順位を付ける
3. 実行計画を作成
4. 実装
5. 評価

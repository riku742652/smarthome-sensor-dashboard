# Core Beliefs - 基本信念

## プロジェクトの価値観

### 1. エージェントファースト

**すべてのコードはエージェントによって生成される**

- 人間は手作業でコードを書かない
- エージェントが理解できる形でドキュメントを書く
- リポジトリ内の情報のみがエージェントの知識ソース

### 2. 計画と実行の分離

**コード実装前に書面化された計画を必ずレビューして承認する**

- 無駄な作業を防ぐ
- アーキテクチャ決定の管理を維持
- 人間の判断を最も効果的に活用

### 3. 段階的開示

**エージェントを最初から膨大な情報で圧倒しない**

- AGENTS.md は目次として機能（~100行）
- 詳細は専門ドキュメントに分離
- 必要に応じて深掘りできる構造

### 4. 機械的な検証

**ルールは文書ではなく、ツールで適用する**

- カスタムリンター
- 構造テスト
- CI/CDパイプライン
- 自動化された品質チェック

### 5. 境界での検証

**すべての外部データは境界で検証する**

- "YOLO スタイル"のデータアクセス禁止
- 型付きスキーマを使用（Zod等）
- 信頼できないデータは仮定しない

### 6. 明示的な依存関係

**暗黙的な依存や循環依存を許可しない**

- 階層化アーキテクチャ
- 依存方向は下方向のみ
- 横断的関心事はプロバイダー経由

### 7. 継続的なガベージコレクション

**技術的負債は高金利のローン**

- 定期的なクリーンアップ
- 負債は記録し優先順位付け
- 複利で膨らむ前に返済

### 8. エージェントの認識可能性

**コンテキスト内にないものは存在しない**

- すべてリポジトリにバージョン管理
- Google DocsやSlackではなくマークダウン
- エージェントが直接アクセスできる形式

## 開発原則

### シンプルさを優先

- 複雑な抽象化より単純な実装
- 過度な最適化は避ける
- 必要になってから追加する

### 退屈な技術を選ぶ

- エージェントがモデル化しやすい
- トレーニングデータに多く存在
- APIが安定している
- ドキュメントが豊富

### 保守性を重視

- 将来のエージェント実行を見据える
- コードベースの一貫性
- 可読性とパターンの統一

### テストは投資

- テストは将来の変更を安全にする
- 回帰を防ぐ
- ドキュメントとしても機能

## 非機能要件

### パフォーマンス

- バンドルサイズ: 初期ロード < 500KB
- Time to Interactive: < 3秒
- API レスポンス: < 2秒

### セキュリティ

- APIキーは環境変数のみ
- すべての外部入力を検証
- 内部情報の漏洩を防ぐ

### 品質

- テストカバレッジ: 80%以上
- TypeScript strict mode
- ゼロリントエラー
